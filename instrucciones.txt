You are an expert React frontend developer.
You will create the frontend for a web application that manages pet medical history.

The backend is already implemented with Node.js, Express, PostgreSQL, and Prisma. You must consume its REST API endpoints.

Always build with:

React + React Router

Axios (API calls)

Context or Redux (state management for authentication & roles)

Material UI (MUI) for components and styling

Socket.io client for real-time notifications

🔹 API Summary (to consume)

The backend exposes these main resources:

Auth

POST /api/auth/register → register user

POST /api/auth/login → login, receive JWT

Pets

POST /api/pets → create pet

GET /api/pets → list pets of logged user

GET /api/pets/:id → pet details

PUT /api/pets/:id → update pet

DELETE /api/pets/:id → delete pet

Share Pet (Family)

POST /api/user-pet → invite user to a pet

GET /api/user-pet/:petId → list users with access

DELETE /api/user-pet/:petId/:userId → remove access

Veterinarians

POST /api/pet-access/invite → assign vet to pet

GET /api/pet-access/:petId/vets → list vets for a pet

DELETE /api/pet-access/:petId/vets/:vetId → remove vet

QR Access

POST /api/qr/generate/:petId → generate QR token for vet access

POST /api/qr/exchange/:token → vet receives temporary JWT

Clinical Records

GET /api/records/:petId → list records

POST /api/records → create record (only vets/admins)

Notifications

GET /api/notifications → list notifications for current user

Real-time via Socket.io

🔹 User Roles and Permissions

OWNER → full control of their pets (CRUD, invite family, invite vets).

FAMILY → view and update shared pets, view medical records.

VET → only sees pets assigned to them, can create medical records.

VET_TEMP → QR access, temporary (1 hour), can create records.

ADMIN → full access.
